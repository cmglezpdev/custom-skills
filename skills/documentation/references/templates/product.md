# Product/PRD Template

Use for product requirements, feature specs, and business-facing documentation. Place in `product/`.

```md
---
title: '<Feature/Product name>'
summary: '<What problem this solves>'
status: draft
owner: '<product-team>'
last_reviewed: YYYY-MM-DD
source_paths: []
---

# [Feature/Product name]

## Problem Statement

What problem exists and why it matters.

## Target Users

Who will use this and in what context.

## Success Metrics

How we measure if this works:

- Metric 1
- Metric 2

## Requirements

### Must Have

- Requirement 1
- Requirement 2

### Nice to Have

- Requirement 3

## Out of Scope

What this feature explicitly does NOT include.

## User Flow

Step-by-step description of the main user flow.

1. User does X.
2. System responds with Y.
3. User sees Z.

## Dependencies

- External services
- Other features that must exist first

## Security & Permissions

Who can access this feature and under what conditions.

## Open Questions

- Question 1
- Question 2

## Change Log

- YYYY-MM-DD: short change entry
```
