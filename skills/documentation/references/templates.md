# Documentation Templates

Each template is in its own file under `references/templates/`. Read the one that matches your document type from the decision guide.

All templates use the same frontmatter standard. Adjust `owner` and `source_paths` for your project.

| Template | File | Use for |
|----------|------|---------|
| Feature | `templates/feature.md` | Features, capabilities, domain logic |
| ADR | `templates/adr.md` | Architecture decisions with trade-offs |
| Workflow | `templates/workflow.md` | Step-by-step engineering or operations processes |
| Runbook | `templates/runbook.md` | Incident response, troubleshooting guides |
| Postmortem | `templates/postmortem.md` | Post-incident reports |
| Product/PRD | `templates/product.md` | Product requirements, business-facing specs |
| Onboarding | `templates/onboarding.md` | Developer onboarding guides |
